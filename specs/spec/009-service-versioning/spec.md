# Feature 009: Service-Level Versioning

> **⚠️ MERGED INTO SPEC 007**
>
> This spec has been merged into [Feature 007: Service Versioning & Deployment](../007-app-versioning/spec.md).
>
> The original spec proposed a separate "service-level" versioning system alongside "app-level" versioning.
> Since we version `Service` configs directly (not `App` groups), there is no app-vs-service
> versioning distinction. Everything is in spec 007:
>
> - **ServiceVersion** — config snapshots for each Service (spec 007)
> - **AppSnapshot** — captures all service versions in an app at a point in time (spec 007)
>
> This file is kept for reference only. All implementation should follow spec 007.
